Getting Familiar with SQLite

Estimated Time: 1 hour

A database is a way of collecting and organizing data so that you can easily find and combine desired pieces of data. A relational database stores information about both data and how it is related. Data and relationships are represented in flat, two-dimensional tables that preserve relational structuring. A database table is a set of values organized into columns and rows. A database row represents a single data item in a table, and is sometimes called a tuple or a record. A field is a single item at one column and one row position.

To access, clean and analyze data that is stored in a relational database, we need to know SQL. It's super easy to learn, and is one of the foundational tools of data science. As mentioned in the start of this lesson, while a SQL standard does exist, almost no relational database system implements the standard exactly. In this course, we'll be using SQLite. SQLite understands most of the standard SQL. It's probably a good idea to bookmark SQL as Understood by SQLite for reference. It describes the parts of the SQL standard that are and are not supported in SQLite, SQL keywords, and SQL syntax.

Installing SQLite

In this assignment you're going to install and setup SQLite on your system and use it to perform some simple data manipulation. You are going to create a table, add some data, perform some basic queries and save the results to a CSV file.

  • Check to see if you already have sqlite3 installed. (If you're on a Mac, you probably do.) For this course, you'll need SQLite ≥ 3.8.
$ sqlite3 --version
3.8.2 2013-12-06 14:53:30 27392118af4c38c5203a04b8013e1afdb1cebd0d
  • If you don't see a version greater than 3.1 and you're on a Mac, go to the SQLite download page and download sqlite-autoconf-*.tar.gz from the source code section. After it's downloaded, issue the following commands in the Terminal:
$ tar xvfz sqlite-autoconf-3071502.tar.gz
$ cd sqlite-autoconf-3071502
$ ./configure --prefix=/usr/local
$ make
$ make install

If you aren't on a Mac, check out the Tutorialspoint instructions for installing SQLite.

Double check that the install has been successful by running sqlite3 --version again.

Create your database

Once you have SQLite installed run the following command:

$ sqlite3 getting_started.db

The above command will create an empty database called 'getting_started' and take you to a command prompt to work with the database. Easy enough, right?

At the command prompt you can enter SQL commands to control the database. You can also enter commands which are specific to SQLite to tell you information about the state of the database. Each of these commands starts with a dot (.). Try them out by listing the tables, indices, and schemas in the database (don't worry that they're all empty; we haven't created anything yet after all!):

> .tables
> .indices
> .schema

Create basic tables

To create a new table, you'll need to specify a table name, the column names, and the data type of each column.

Try entering the following statement to create a new table:

> CREATE TABLE cities (name text, state text);

There's quite a lot of information contained in the above statement, so let's it down into its constituent parts:

  • CREATE TABLE is the SQL command for creating a new table
  • cities is the name of the table we want to create
  • The opening parenthesis indicates the start of the list of columns which the table will hold
    • The different columns will be separated by commas
  • name text is an example of a column definition
    • The column name will be name
    • The data type of the column is text (i.e. the column will hold a string of text)

Now you should add some rows to the table. The following code adds rows for a number of US cities:

> INSERT INTO cities (name, state) VALUES
    ('New York City', 'NY'),
    ('Boston', 'MA'),
    ('Chicago', 'IL'),
    ('Miami', 'FL'),
    ('Dallas', 'TX'),
    ('Seattle', 'WA'),
    ('Portland', 'OR'),
    ('San Francisco', 'CA'),
    ('Los Angeles', 'CA');

New rows are added using the INSERT INTO command. You specify that the rows want to be added to the cities table. The (name, state) section indicates which columns you are adding data to, and the order in which you will supply the data. In this case the name of the city comes first, and the state the city belongs to comes second.

VALUES indicates the start of the list of values which you are going to insert. Each row is supplied as a list of values contained in parentheses. So for example here you create a row where the name column contains the value "New York City", and the state column contains the value "NY".

Try it!

  • Create another table called 'weather' with the columns 'city', 'year', 'warm_month', 'cold_month', and 'average_high' which should be text, integer, text, text and integer types respectively.

  • Add the following data to the table:

      City            Year    Warm Month  Cold Month  Average High
  New York City   2013    July        January     62
  Boston          2013    July        January     59
  Chicago         2013    July        January     59
  Miami           2013    August      January     84
  Dallas          2013    July        January     77
  Seattle         2013    July        January     61
  Portland        2013    July        December    63
  San Francisco   2013    September   December    64
  Los Angeles     2013    September   December    75

  • Where does this data come from, you might (and should) ask. Here.

  • Try listing the tables and schema in the database with .tables and .schema. You should see the two tables that we just created.

Write queries on your data

Your database would not be very useful if your could add rows, and not retrieve them. The select statement is used to perform queries on the database. Try running the following examples:

SELECT * FROM cities;

Here you ask SQLite to query for all of the columns (*) in the cities table. You should see each of the rows you create earlier printed back.

SELECT name, state FROM cities;

The is a more explicit version of the previous query. You describe exactly which columns should be returned from the table. It is generally preferable to use this more explicit syntax, as it fixes the order in which the columns are returned, and encourages you to only return as much data is necessary.

SELECT name, state FROM cities WHERE state='CA';

Next you make the query more specific using the WHERE clause. In this case we look for cities which are in California, by returning the rows where the state column is equal to CA.

SELECT name FROM cities WHERE name LIKE '%le%';

You don't just have to filter rows based on equality. You can also use other comparators; for example SELECT name FROM people WHERE age >= 21; would find the over-21s in a table. In the above example we use the LIKE keyword to find the city names which contain the substring le. The % characters act as a wildcard, indicating that any number of characters can come before or after the substring.

SELECT name FROM cities LIMIT 2 OFFSET 3;

Sometime you don't want to deal with all of your data at once. This example introduces the LIMIT and OFFSET clauses. The LIMIT clause limits the number of rows that are returned by the query. The OFFSET clause ignores the first N rows which match the query, giving you an offset into the data. So this query will return the fourth and fifth cities only.

SELECT COUNT(*) FROM cities WHERE name LIKE 'San%' AND state='CA';

The COUNT function is used to return the number of rows which match a query. So in this example you count the number of cities in California which start with the string "San". Notice how we can logically combine two WHERE clauses using AND, OR and NOT.

SQLite contains a number of additional built-in functions which you may find useful. For example:

  • SUM() finds the sum of the results
  • AVG() finds the average of the results
  • MIN() finds the lowest result
  • MAX() finds the highest result

Try it!

Write queries to find out the following pieces of information.

  • How many rows are in the weather table?
  • What cities were hottest in July in 2013?
  • What cities were hottest in July and not coldest in January?
  • The first two cities which were coldest in January

Update your data

To change the values held in a table you use the UPDATE command. Try running the following example:

UPDATE cities SET state='Californ-I-A' WHERE state='CA';

Here we update the cities table, changing the state name for the cities in California.

Try it!

  • Use an UPDATE statement to undo the meddling from above, changing "Californ-I-A" back to "CA".

Deleting data

To delete data from a table you use the DELETE FROM command. For example, to delete the Californian cities you would run the following statement:

DELETE FROM cities where state='CA';

Try it!

  • Delete all of the remaining rows from the cities table
  • Re-add the original data using the INSERT INTO command from above

Export your data to CSV

In some situations, we might want to export a table in a database or the results of a query on a database, to a CSV.

Let's try exporting the cities table to a CSV names cities.csv, with the column names as the header:

> .mode csv
> .headers on
> .output cities.csv
> select * from cities;
> .output stdout

First you switch the output mode to CSV, and turn the column headers on. You then tell SQLite to output any statements to the cities.csv file using the .output command. Next you make the query to select all of the rows from the cities table. Then finally the output back is redirected back to the command line.

Import data from a CSV

In other situations, you'll want to take a CSV and import it into a table in a relational database.

Here's an example of how to load the cities.csv file we just created into a new table in our getting_started database. Note that you have to create the new table before loading the csv into it, otherwise the CSV will have nowhere to go.

> create table cities_copy (name text, state text);
> .separator ","
> .import cities.csv cities_copy

First you have to create the new table before loading the csv into it, otherwise the CSV will have nowhere to go. Then you set the comma separator, and tell SQLite to import the data into the table using the .import command.

  • To make sure that it worked, run the .tables command one more time.
